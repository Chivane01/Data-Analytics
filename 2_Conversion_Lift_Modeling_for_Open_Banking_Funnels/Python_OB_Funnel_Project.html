🧠 Project: Conversion Lift Modeling for Open Banking Funnels¶

This project simulates and analyzes a digital banking user journey — from sign-up to transaction — to understand user drop-offs and quantify the impact of improving funnel steps.

📘 Project Context: Why This Matters¶

You're part of the data team at a digital banking platform. Product managers noticed that while many users sign up, only a fraction complete a transaction. The job is to:

  • Simulate a realistic 4-stage user funnel
  • Identify the stage with the biggest drop-off
  • Model "what-if" improvements to evaluate business impact

🟦 Step 1: Simulate Funnel Data¶

Simulated 1,000 users through these 4 funnel steps:

  1. Sign-Up
  2. KYC Completion
  3. Account Linking
  4. Transaction Completion

Each step has a probabilistic drop-off modeled via random sampling.

# ✅ STEP 1: Simulate Open Banking User Funnel Data

# Import required libraries
import pandas as pd
import numpy as np
import random
from datetime import datetime, timedelta

# Set seed for consistent results every time you run
np.random.seed(42)

# Create 1000 fake users
num_users = 1000
user_ids = [f"U{str(i).zfill(4)}" for i in range(1, num_users + 1)]

# Generate random signup dates
start_date = datetime(2023, 1, 1)
signup_dates = [start_date + timedelta(days=np.random.randint(0, 180)) for _ in user_ids]

# Function to simulate whether a user completes the next step
def random_step(previous_step, probability):
    return [
        1 if previous_step[i] == 1 and random.random() < probability else 0
        for i in range(len(previous_step))
    ]

# Funnel simulation
step_signup = [1] * num_users                    # All users sign up
step_kyc = random_step(step_signup, 0.85)        # 85% complete KYC
step_link = random_step(step_kyc, 0.75)          # 75% link account
step_txn = random_step(step_link, 0.60)          # 60% make a transaction

# Create a DataFrame
df = pd.DataFrame({
    "User_ID": user_ids,
    "Signup_Date": signup_dates,
    "KYC_Completed": step_kyc,
    "Account_Linked": step_link,
    "Transaction_Completed": step_txn
})

# STEP 2: Analyze funnel metrics

# Total user counts
total_users = len(df)
kyc_users = df["KYC_Completed"].sum()
linked_users = df["Account_Linked"].sum()
txn_users = df["Transaction_Completed"].sum()

# Conversion rates
conv_signup_to_kyc = kyc_users / total_users
conv_kyc_to_link = linked_users / kyc_users if kyc_users else 0

# Simulate a 10% drop-off improvement in final step

# Actual conversion from Link → Transaction
actual_conversion = df[df["Account_Linked"] == 1]["Transaction_Completed"].mean()
original_dropoff = 1 - actual_conversion
improved_dropoff = original_dropoff * 0.90  # reduce by 10%
improved_conversion = 1 - improved_dropoff

# Projected transaction count
projected_txn = int(total_users * conv_signup_to_kyc * conv_kyc_to_link * improved_conversion)
additional_txn = projected_txn - txn_users

# Output results
print("\n📈 Drop-Off Optimization Simulation")
print(f"Original Drop-Off (Link → Transaction): {original_dropoff:.2%}")
print(f"Improved Drop-Off (10% better): {improved_dropoff:.2%}")
print(f"Projected Transactions: {projected_txn}")
print(f"Additional Conversions Gained: {additional_txn}")

# STEP 6: Simulate improving drop-off at each funnel step (by 10%)

# Function to simulate adjusted transactions given improved drop-off
def simulate_projected_txns(drop1, drop2, drop3, improve="none"):
    # Simulate 10% better drop-off at a selected stage
    if improve == "signup_kyc":
        drop1 *= 0.90
    elif improve == "kyc_link":
        drop2 *= 0.90
    elif improve == "link_txn":
        drop3 *= 0.90

    # Recalculate conversion rates
    conv1 = 1 - drop1
    conv2 = 1 - drop2
    conv3 = 1 - drop3

    # Projected transaction count
    return int(total_users * conv1 * conv2 * conv3)

# Step-specific drop-off rates
drop1 = drop_signup_to_kyc
drop2 = drop_kyc_to_link
drop3 = drop_link_to_txn

# Run simulations
sim1 = simulate_projected_txns(drop1, drop2, drop3, improve="signup_kyc")
sim2 = simulate_projected_txns(drop1, drop2, drop3, improve="kyc_link")
sim3 = simulate_projected_txns(drop1, drop2, drop3, improve="link_txn")

# Store results
scenario_results = {
    "Original Transactions": txn_users,
    "Improve SignUp → KYC": sim1,
    "Improve KYC → Link": sim2,
    "Improve Link → Transaction": sim3
}
